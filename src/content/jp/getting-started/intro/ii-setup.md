はじめに — はじめに: セットアップ

Cursor Notes (またはビューア) をこのリポジトリに接続し、サイドバーが **`src/content`** をロードするようにします。

## 1. ブランチ

このリポジトリは **`main`** を追跡します。デフォルトのブランチが異なる場合は、代わりに設定でそのブランチを使用してください。

## 2. コンテンツのパス

コンテンツルートを次のように設定します。

```text
src/content
```

したがって、アプリは **`src/content/_meta.json`** とその下のすべてのトピック フォルダー (**`getting-started/intro/`** などのネストされたフォルダーを含む) を読み込みます。

## 3. Cursor の GitHub 設定

1. [メモ] メニューから **GitHub** 設定を開きます。
2. **所有者** / **リポジトリ** を **`origin`** と一致するように設定します。
3. **ブランチ** を **`main`** に設定します (リポジトリでまだ使用している場合にのみ **`master`** を試してください)。
4. リポジトリがプライベートの場合は、リポジトリ読み取りアクセス権を持つ **個人アクセス トークン**を貼り付けます。

### プライベートおよびパブリック リポジトリ

メモが **プライベート** リポジトリに存在する場合 (またはプライベート リポジトリとパブリック リポジトリを切り替える場合)、GitHub からファイルをロードするものはすべて、プロフィール/電子メールだけでなく **リポジトリ アクセス**が必要です。

- **GitHub でサインイン** の **同意画面** (「プライベートおよびパブリック リポジトリ」) は、統合で OAuth **`repo`** スコープを使用する場合に予期され、リポジトリのコンテンツに対して GitHub API を呼び出すことができます。
- **最小権限の代替:** ログイン スコープを最小限に抑え、選択したリポジトリで **コンテンツ: 読み取り専用** の **きめ細かい PAT** を使用します。これは、従来の **`repo`** OAuth スコープよりも狭いです。

### OAuth スコープ (独自の GitHub ログインを構築する場合)

サーバーが OAuth (クラシック スコープ) 経由で **プライベート** リポジトリ ファイルを読み取る必要がある場合は、ID フィールドに加えて **`repo`** (**`read:user user:email repo`** など) をリクエストします。

Auth.js / NextAuth スタイルのプロバイダー スニペット:

```ts
GitHub({
  authorization: {
    params: {
      scope: "read:user user:email repo",
    },
  },
}),
```

クラシック **`repo`** は広範囲にわたります (そのアカウントに対する事実上完全なリポジトリ アクセス)。トークンの発行を制御する場合は、**きめ細かい PAT** または **コンテンツ: 読み取り専用** の **GitHub アプリ** を優先します。

## 4. リフレッシュ

新しいフォルダーまたはファイルをプッシュした後、ノートを更新または再同期して、ツリーを更新します。
