Rust — パート III: Rustling で学ぶ

**[Rustlings](50)** は公式の実践的な演習トラックです。壊れた小さなプログラムをコンパイルしてテストに合格するまで修正します。これらのメモや [Rust プログラミング言語](51) の本と**併用**してください。単独で読むのは、入力して意図的に何かを壊すよりも時間がかかります。

## 1. カサカサとは何か

| | |
|---|---|
| **形式** |トピックごとにグループ化された `exercises/` の下の多数の小さな演習 |
| **あなたの仕事** | **`TODO`** / **`todo!()`** を見つけてファイルを編集し、**`cargo`** を幸せにしてください |
| **ツール** | **`rustlings`** CLI は、賢明な順序で演習を進めます。
| **目標** | **`struct`**、**`impl`**、**`match`**、所有権、イテレータなどのマッスル メモリ |

Rustlings は、パート I ～ II や書籍に代わるものではありません**。それらを**強化するものです**。パート I (組織 + 所有権) とパート II (**`cargo`**) を終えると、すぐに始めることができます。

## 2. 前提条件

1. **Rust ツールチェーン** — [rust-lang.org/tools/install](52) (`rustup`、`cargo`、`rustc`)。
2. **`cargo build` はご使用のマシンで動作します**。**Windows** では、最初に **MSVC C++ ビルド ツール** をインストールします (**パート I**、§1 を参照)。
3. **エディタ** — VS Code (またはそれをサポートするエディタ) の [rust-analyzer](53) により、エラーがインラインで表示されます。

Rustlings をインストールする前にアップデートしてください:

```text
rustup update
```

## 3. インストールと初期化

演習フォルダーを作成する任意のディレクトリから:

```text
cargo install rustlings
rustlings init
cd rustlings
rustlings
```

インストールが失敗した場合は、次のことを試してください。

```text
cargo install rustlings --locked
```

**`rustlings` が見つかりませんか?** Cargo はバイナリを **`~/.cargo/bin`** (Linux/macOS) または **`%USERPROFILE%\.cargo\bin`** (Windows) に配置します。そのディレクトリを **PATH** に追加し、**新しい** ターミナルを開きます。

**Windows** では、**[Windows ターミナル](54)** を使用して、最高のウォッチモード エクスペリエンスを実現します (**WSL** を使用している場合はこれを含みます)。

## 4. セッションの仕組み

1. **`rustlings`** は **ウォッチ モード**を開始します。現在のエクササイズが実行され、**`exercises/`** でファイルを保存すると再実行されます。
2. コンパイラ エラーを読み取り、演習ファイルを編集し、再度保存します。
3. 演習が終了すると、Rustlings は次の演習に進みます。

ウォッチモードで便利なキー:

|キー |アクション |
|-----|----------|
| **`h`** |現在の演習のヒント |
| **`l`** |インタラクティブ **エクササイズ リスト** (完了 vs 保留、ジャンプ、リセット) |
| **`r`** |現在のエクササイズを再実行します (**`--manual-run`** が設定されている場合にも使用されます)。

ファイルの監視が失敗した場合 (一部の VM/コンテナー)、次を使用します。

```text
rustlings --manual-run
```

保存するたびに **`r`** を押します。

各トピック フォルダーにはリンク付きの **`README.md`** があります。そのセクションの演習に入る前にざっと読んでください。

## 5. 練習する内容 (およびそれを取り上げた場所)

|カサカサ音エリア（通常） |補強 |
|--------------------------|---------------|
|変数、関数 |パート I — プログラムの形状 |
| **`struct`**、**`enum`**、**`match`** |パート I — 組織 |
|所有権、借用、スライス |パート I — 所有権 |
| **`struct`/`enum` メソッド**、モジュール |パート I — **`impl`**、モジュール |
|コレクション、**`String`** |本ch. 8 + 演習 |
|エラー処理 **`Result`** |パート I — **`Result`**、書籍 ch. 9 |
|ジェネリック、**`trait`**、ライフタイム |本ch. 10–11 (基本の後にさらに深く進みます) |
|テスト、**`clippy`**、**`macro`s** |パート II — **`cargo test`**、品質の習慣 |

演習で **木箱** または **`Cargo.toml`** について言及している場合は、**パート II** [貨物と共有可能な木箱](ii-cargo-and-shareable-crates.md) をクロスチェックしてください。

## 6. 推奨される学習順序

```text
Part I (this track)     →  skim organization + ownership
       ↓
rustlings init          →  run `rustlings` daily in short blocks
       ↓
The Book (in parallel)  →  same chapters as the exercise topic
       ↓
Part II (Cargo)         →  when exercises touch modules / deps
       ↓
Your own `cargo new`    →  tiny bin + lib project from scratch
```

練習中の **オフライン ドキュメント**:

```text
rustup doc --book
rustup doc --std
```

## 7. 行き詰まったとき

1. 監視モードで **`h`** を押すと内蔵ヒントが表示されます。
2. **`exercises/...`** フォルダ内のトピック **`README.md`** をもう一度読みます。
3. [カサカサ音Q&Aディスカッション](55)を検索します。
4. そのトピックについて [Rust Book](56) の章と比較します。Rustlings は、本の代わりにではなく**並行して**実行されるように設計されています。

難しいと感じるからといってエクササイズを永久にスキップしないでください。**`l`** を使用して先にジャンプして戻ってきてください。クリーンなファイルが必要な場合は、リストの **`r`** を使用して演習をリセットします。

## 8. カサカサ音のあと

- 小さな **`cargo new`** プロジェクト (CLI ツール、パーサー、ゲーム ループ) を構築します。
- **ワークスペース**を公開または分割する場合は、**パート II** をもう一度お読みください。
- オープンソースの Rust クレートに貢献するか、他の人に教えたい場合は [コミュニティ演習](57) を追加してください。
