APIゲートウェイ — 操作とトラブルシューティング

**アクセス ログ**、**アップストリームの健全性**、CDN、ゲートウェイ、サービス全体にわたる明確な**タイムアウト**の調整を使用してゲートウェイを運用します。

## 1. 注目すべき指標

|メトリック |信号 |
|----------|----------|
| **4xx レート** |認証の設定ミス、不正なクライアント |
| **5xx レート** |アップストリームまたはゲートウェイの過負荷 |
| **レイテンシ p50/p99** |ゲートウェイのオーバーヘッドとサービスの遅さ |
| **429 レート** |レート制限の機能 - しきい値の調整 |
| **統合エラー** | Lambda/ALB に到達できません |

**ルート**と**APIバージョン**ごとにダッシュボードを分割します。

## 2. 失敗したリクエストをデバッグする

```text
1. Reproduce with curl -v (include Authorization if needed)
2. Check gateway access log — was upstream called?
3. If 401/403 → auth plugin / JWT claims
4. If 429 → rate limit key / Redis counter
5. If 502/504 → upstream health, timeout, security group
6. Compare direct-to-upstream (VPN) vs via gateway
```

```bash
curl -v https://api.example.com/api/v1/health \
  -H "Authorization: Bearer $TOKEN"
```

## 3. よくある失敗

|症状 |考えられる原因 |修正 |
|----------|--------------|-----|
| **502 不正なゲートウェイ** |アップストリームのダウン、間違ったポート、VPC リンクの破損 |ヘルスチェック対象者 |
| **504 ゲートウェイ タイムアウト** |サービスが遅い。ゲートウェイのタイムアウトが低すぎます |タイムアウトを増やすかサービスを修正してください。
| **有効なユーザーの 401** |間違った JWKS、クロック スキュー、`aud` の不一致 | NTP を同期します。発行者の構成を修正する |
| **ブラウザでの CORS エラー** |ゲートウェイが見つかりません `Access-Control-*` |ゲートウェイに CORS プラグインを追加する |
| **二重スラッシュ / 404** | `strip_path` 不一致 |ゲートウェイの書き換えをサービス マウントと調整する |
| **Postman では動作しますが、本番環境では失敗します** |別のホスト、WAF ルールがありません |ヘッダーとパスを比較する |

## 4. CDN + ゲートウェイのインシデント

|症状 |チェック |
|----------|----------|
| API が間違ったユーザーに対してキャッシュされました | `/api/*` の CDN キャッシュ — バイパスまたは `no-store` |
|断続的 504 | CDN タイムアウト < ゲートウェイ < サービス — 整列 |
| SSL エラー | CDN とゲートウェイのホスト名の不一致に関する証明書 |

完全なトポロジ: [CDN と API ゲートウェイを一緒に](../cdn/viii-cdn-and-api-gateway-together.md)。

## 5. デプロイメントとロールバック

- **青/緑のアップストリーム** — スイッチ ゲートウェイ ターゲットの重み
- **CI でのルート構成** — lint OpenAPI、ドライラン適用
- サービスの **機能フラグ** — ゲートウェイ ルートは安定しています。重みを介したカナリア

`/api/v2` の展開中は **下位互換性** `/api/v1` を維持します。

## 6. セキュリティ運用

- API キーをスケジュールに従ってローテーションします。未使用のコンシューマを監査する
- WAF ブロックを確認する — 誤検知と実際の攻撃
- アプリサーバーだけでなく、パブリックゲートウェイ表面の侵入テスト

## 7. チェックリスト

- [ ] ルートは OpenAPI / コントラクト テストと一致します
- [ ] 認証はすべての非パブリック パスに適用されます
- [ ] 層ごとのレート制限が文書化されています
- [ ] CDN → ゲートウェイ → サービスで調整されたタイムアウト
- [ ] SIEM またはログ アグリゲータに送信されるアクセス ログ
- [ ] サービスを介したゲートウェイからのトレース ID
- [ ] `/health` および `/ready` は重いプラグインから除外されます
- [ ] CDN バイパスまたは認証済み API の `no-store`
