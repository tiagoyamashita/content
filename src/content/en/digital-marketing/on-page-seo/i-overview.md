On-page SEO — overview
**On-page SEO** is everything you control on your own pages so search engines can **crawl**, **index**, and **rank** them for queries that match user intent.

Off-page factors (backlinks) live in [Link building](../link-building/i-overview.md). This submenu is the foundation — fix on-page before heavy outreach.

## 1. Crawl → index → rank

```text
Crawl    →  bot discovers URL (links, sitemap)
Index    →  URL eligible to appear in search results
Rank     →  position for a query (many signals, including links)
```

A page can be **crawled but not indexed** (thin/duplicate/low value) or **indexed but not ranking** (competition, weak content, weak links).

## 2. Page-level signals (summary)

| Signal | What you control |
|--------|------------------|
| **Title & meta** | Snippet in search results — affects CTR |
| **Headings & body** | Topic clarity and depth |
| **URL** | Short, readable, stable |
| **Internal links** | Which pages get authority and discovery |
| **Technical health** | Speed, mobile, HTTPS, no accidental blocks |

## 3. Intent beats keywords

Modern SEO targets **search intent** — what the user wants when they type a query — not keyword density.

| Intent | Example query | Page type |
|--------|---------------|-----------|
| **Informational** | "what is link building" | Guide, glossary |
| **Commercial** | "best email tool for startups" | Comparison, listicle |
| **Transactional** | "buy domain name" | Product, pricing |
| **Navigational** | "yourbrand login" | App login, brand homepage |

One primary intent per URL. See [Keywords & search intent](iii-keywords-and-search-intent.md).

## 4. Prerequisites checklist

| Check | If failing |
|-------|------------|
| Site live on **HTTPS** | Fix before anything else |
| Important pages not `noindex` | Remove accidental blocks |
| **Google Search Console** verified | Free index and query data |
| Core pages linked from nav or homepage | Reduce orphan pages |

Full list: [Technical checklist](v-technical-checklist.md).
