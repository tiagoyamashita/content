Content strategy — overview
**Content strategy** is the plan for **what** you publish, **why**, and **how** it supports acquisition and trust — not a calendar of filler posts.

Good content feeds [On-page SEO](../on-page-seo/i-overview.md) and [Link building](../link-building/i-overview.md). Bad content wastes crawl budget and brand.

## 1. Content marketing vs blogging

| Blogging (default) | Content strategy |
|--------------------|------------------|
| "We should post weekly" | Each piece maps to ICP pain + funnel stage |
| Random topics | Pillars aligned to search + sales |
| Publish and forget | Update, distribute, measure |

Start from [Audience & positioning](../foundations/iii-audience-and-positioning.md) — who reads this and what should they do next?

## 2. Content–funnel alignment

| Funnel stage | Content examples |
|--------------|------------------|
| Awareness | How-to guides, glossary, industry trends |
| Consideration | Comparisons, case studies, webinars |
| Conversion | Product tours, ROI calculators, pricing FAQs |
| Retention | Advanced tutorials, release notes, community highlights |

## 3. Minimum viable content (new site)

Before volume, ship **cornerstone** pages:

```text
Homepage + clear value prop
1 pillar guide (best in class for one query cluster)
Pricing / signup (if applicable)
About / trust (team, security, contact)
2–3 supporting articles linking to pillar
```

Then expand clusters — see [Pillars & topic clusters](ii-pillars-and-topic-clusters.md).

## 4. Quality bar

Ask before publish:

| Question | |
|----------|---|
| Does this answer a real question better than page 1 today? | |
| Would our ICP share or bookmark it? | |
| Is there a clear next step (CTA + internal link)? | |
| Can we maintain/update it in 12 months? | |
