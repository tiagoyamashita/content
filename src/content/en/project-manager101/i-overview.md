Project Manager 101 — overview
Project management turns an agreed objective into a coordinated delivery plan. A project manager aligns scope, people, time, cost, risk, and communication so a temporary initiative can reach a clear finish.

## Project manager vs product manager

| Project manager | Product manager |
|-----------------|-----------------|
| Owns how and when an initiative is delivered | Owns what product outcome to pursue and why |
| Manages scope, schedule, dependencies, and risks | Manages customer problems, priorities, and product results |
| Usually works toward a defined end date | Usually manages an ongoing product lifecycle |
| Measures delivery predictability and project outcomes | Measures user and business outcomes |

The roles collaborate, and one person may cover both in a small organization, but the decisions are different.

## A practical lifecycle

```text
Initiate → Plan → Execute and monitor → Deliver → Close and learn
```

Planning is not a one-time event. Update the plan when evidence changes, while keeping the objective and decision history visible.

## Core habits

1. Define success before creating a schedule.
2. Make ownership and dependencies explicit.
3. Separate facts, assumptions, risks, and decisions.
4. Escalate early with options, not surprises.
5. Control changes instead of pretending the original plan is fixed.
6. Close the project with acceptance, handover, and lessons learned.
